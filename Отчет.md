# 📊 Отчёт: Гибридная рекомендательная система на основе коллаборативной фильтрации
## Постановка задачи
*Цель проекта:* Реализация гибридной рекомендательной системы, объединяющей два подхода коллаборативной фильтрации:
User-based: рекомендации на основе похожих пользователей
Item-based: рекомендации на основе похожих фильмов
*Задачи:*
1. Реализовать оба подхода с нуля (без библиотек ML)
2. Разработать механизм объединения результатов с усилением пересечений
3. Оценить качество через метрики: Hit Rate@K, Precision@K, Recall@K
4. Провести эксперименты:
    - Скорость работы на датасете 335 МБ
    - Оптимизация производительности
    - Решение проблемы холодного старта
*Датасеты:*
- Основной: MovieLens 1M (1 млн оценок, ~6 тыс. пользователей, ~4 тыс. фильмов)
- Для масштабирования: MovieLens Latest Full (33 млн оценок, 330 тыс. пользователей, 86 тыс. фильмов)

Холодный старт: новые пользователи/фильмы без истории взаимодействий
## Подход к реализации
### Архитектура системы
Данные в датасетах -> User-based -> Item-based -> Слияние -> Вывод рекомендаций
### Ключевые компоненты
#### Косинусное сходство
- Для пользователей: сравнение векторов оценок фильмов
- Для фильмов: сравнение векторов оценок пользователей
- Учитывает только общие элементы (общие фильмы/пользователи)
#### Гибридное объединение
Формула финальной оценки для фильма i:
Оценка = boost_factor * (a x norm(user_vector))+((1-a) x norm(item_vector)) Если в обоих списках
Где:
- norm — нормализованные оценки (0..1)
- a — баланс между подходами (0.5 = равные веса)
- user_vector — вектор оценок пользователя
- item_vector — вектор оценок фильма
- boost_factor — усиление пересечений (по умолчанию 1.5)

## Эксперименты и оптимизация
### Скорость обработки на большом датасете (335 МБ)
#### Основная Проблема
При загрузке MovieLens Latest Full (33 млн оценок):
- Время построения словарей: ~45 сек
- Расчёт сходства для одного пользователя: ~120 сек (без кэширования)
- Память: ~1.8 ГБ
#### Причины низкой скорости
- Квадратичная сложность: сравнение каждого пользователя со всеми → O(n²)
- Отсутствие кэширования: повторный расчёт сходства для каждой рекомендации
- Интерпретируемый Python: медленнее нативных реализаций
#### Решения для ускорения
- Кэширование сходства (сильно ускорит) - Хранить топ-50 похожих для каждого пользователя/фильма
- Ограничение кандидатов (ускорит, но могут потеряться данные) - Рассматривать только популярные фильмы (>100 оценок)
- Параллелизация (ускорит сильно, но на огромных данных) 
- NumPy векторизация (ускорит средне) - Использование матричных операций, но сложнее реализовать
- Библиотека scipy (ускорит средне) - Использование специальных библиотек для расчётов  

#### Пример оптимизированного кода с scipy

from scipy.sparse import csr_matrix
from sklearn.metrics.pairwise import cosine_similarity

def build_sparse_matrix(user_ratings, all_users, all_items):
    """Построение разреженной матрицы пользователь-фильм"""
    rows, cols, data = [], [], []
    user_idx = {u: i for i, u in enumerate(all_users)}
    item_idx = {i: j for j, i in enumerate(all_items)}
    
    for user, ratings in user_ratings.items():
        for item, rating in ratings.items():
            rows.append(user_idx[user])
            cols.append(item_idx[item])
            data.append(rating)
    
    return csr_matrix((data, (rows, cols)), shape=(len(all_users), len(all_items)))

def fast_user_similarity(sparse_matrix, user_idx):
    """Быстрое вычисление сходства для одного пользователя"""
    user_vector = sparse_matrix[user_idx]
    similarities = cosine_similarity(user_vector, sparse_matrix)
    return similarities.flatten()

*Результат:* расчёт сходства для 330 тыс. пользователей за ~8 сек вместо ~10 часов.
## Решение проблемы холодного старта
### Основная проблема
Новый пользователь без истории оценок → невозможно применить коллаборативную фильтрацию.
### Предложенные решения
*Методы*
1. Выводить топ-10 самых популярных сейчас
2. Спросить 3 любимых жанра → рекомендовать по ним
3. Показать 10 случайных фильмов → собрать первые оценки

### Реализация: двухэтапный подход

def cold_start_recommendations(item_ratings, movie_id_to_title, n_rec=10):
    """Рекомендации для нового пользователя (без истории)"""
    # Этап 1: Топ-20 популярных фильмов (по числу оценок)
    popularity = {mid: len(ratings) for mid, ratings in item_ratings.items()}
    top_popular = sorted(popularity.items(), key=lambda x: x[1], reverse=True)[:20]
    
    # Этап 2: Фильтрация по качеству (средний рейтинг > 3.5)
    quality_filtered = []
    for mid, _ in top_popular:
        avg_rating = np.mean(list(item_ratings[mid].values()))
        if avg_rating >= 3.5:
            quality_filtered.append((mid, avg_rating))
    
    # Возвращаем топ-N
    quality_filtered.sort(key=lambda x: x[1], reverse=True)
    return [mid for mid, _ in quality_filtered[:n_rec]]

### Ключевые результаты
Гибридная система превосходит отдельные подходы благодаря усилению пересечений (boost_factor=1.5)
Оптимальные гиперпараметры:
- alpha = 0.5–0.6 (равный баланс подходов)
- boost_factor = 1.5 (умеренное усиление)
- k_neighbors = 10–15 (баланс качества/скорости)
- Холодный старт решается двухэтапно: популярные фильмы → сбор первых 5–7 оценок → полноценная рекомендация
### Практические рекомендации
1. Использовать кэширование + scipy для датасетов >100 тыс. пользователей
2. Фильтровать фильмы с < 20 оценок перед расчётом сходства
3. Показывать стартовый набор из 10 фильмов разных жанров
4. Добавить штраф за жанровую однородность в топ-10
### Возможные улучшения
- Интеграция контентных признаков (жанры, актёры) для решения холодного старта фильмов
- Временные веса: учитывать свежесть оценок (новые важнее старых)
- Адаптивный boost_factor: динамически менять в зависимости от уверенности системы
- Онлайн-обучение: обновление модели при каждой новой оценке без полного пересчёта
